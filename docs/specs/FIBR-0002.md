**Theme:** make it **structurally impossible to stage financial
data or build output** into this public repo. The `.gitignore`
is the first line (git never offers the file); `gitleaks`
(FIBR-0001) is the backstop for a secret that slips past it.

## Context

The repository is public (security-model A7): it must hold code +
docs only — never a vault, a key, or real statement data. A
developer running the app against a real vault, or building a
release locally, produces exactly the artefacts that must never be
committed: the encrypted `*.db` vault (A1), and PyInstaller /
AppImage / Flatpak / `.dmg` output. This item closes that gap by
covering those paths in `.gitignore`, satisfying the `.gitignore`
half of security-model **T6** ("`.gitignore` excludes
`*.db`/vault/build output").

FIBR-0001 already shipped a partial `.gitignore` (Python bytecode,
`.venv`, OS/editor droppings, `.env` secrets, Claude Code config).
This item **extends** that file — it does not rewrite it — with the
financial-data, tooling-cache, and packaging-output patterns.

`gitleaks` is a **backstop, not the primary control**: it scans
file *content* for secret patterns, but an encrypted vault is
high-entropy ciphertext with no recognisable secret pattern, so
`.gitignore` (which excludes by *path*) is what actually keeps a
vault out. The two are defence-in-depth, not redundant.

## Invariants (testable)

> These `INV-N` are **local to this spec**. Where this spec cites a
> project-wide security invariant it names the doc (e.g.
> "security-model.md T6").
>
> **Test mechanics (applies to INV-1..INV-3).** The test queries each
> pattern with **`git check-ignore --no-index <path>`**. `--no-index`
> is mandatory: a bare `git check-ignore` reports every *tracked* file
> as "not ignored" regardless of the rules, which would make INV-3
> pass vacuously (false assurance). Directory patterns (`build/`,
> `dist/`, `*.egg-info/`, `.flatpak-builder/`) are queried via a path
> *inside* the directory (e.g. `dist/finbreak.exe`), because
> `git check-ignore build` with no trailing slash and no on-disk
> directory reports "not ignored". Convention: **exit 0 = ignored;
> non-zero exit = not ignored.**

- **INV-1 — No financial data can be staged.** The check reports a
  representative vault / database file as ignored from any directory
  in the tree — at minimum `*.db`, `*.db-*`, `*.sqlite`,
  `*.sqlite-*`, `*.sqlite3`, `*.sqlite3-*` (e.g. `finbreak.db`,
  `sub/dir/myvault.sqlite`, and the SQLite runtime sidecars a live
  vault writes next to itself: `finbreak.db-wal`, `finbreak.db-shm`,
  `finbreak.db-journal`). The encrypted vault (security-model A1) —
  and its WAL/journal sidecars, which hold the same vault data — is
  the primary target. Source: security-model.md T6, A1, A7.
- **INV-2 — No build, packaging, or tooling output can be staged.**
  The check reports the build and packaging output as ignored — the
  PyInstaller / AppImage / Flatpak / macOS tool artefacts `build/`,
  `dist/`, and `.flatpak-builder/` directories (queried via a path
  inside each, per Test mechanics) plus the file artefacts `*.dmg`,
  `*.AppImage`, and `*.flatpak`; the setuptools build-metadata
  directory `*.egg-info/` (emitted by `pip install -e .` /
  `python -m build`, not by the four packaging tools); and the
  tooling caches `.pytest_cache/`, `.ruff_cache/`, `.mypy_cache/`.
  (The caches are dev hygiene rather than a T6 data-leak concern, but
  the same `git check-ignore` assertion covers them, so no delivered
  ignore pattern is left untested.) Source: security-model.md T6;
  discovery.md "Distribution"; design.md "Packaging".
- **INV-3 — Real source is still tracked (no over-broad ignore).**
  Under `--no-index` (so the check reflects the *rules*, not the
  already-tracked state — see Test mechanics), the check reports a
  **non-zero** exit (i.e. NOT ignored) for real project files —
  `src/finbreak/__init__.py`, `docs/design.md`, `pyproject.toml`,
  `tests/test_smoke.py`. A `.gitignore` that hides source is as much
  a defect as one that leaks data. Source: this spec (guard against
  pattern over-reach).
- **INV-4 — The secret-leak backstop still runs green.** After the
  ignore set is extended, `gitleaks` (via `scripts/ci-local.sh`,
  FIBR-0001 INV-1) still exits 0 on the working tree. Source:
  security-model.md T6, INV-6; FIBR-0001 INV-1/INV-5.
  *Design note (not itself a testable invariant):* `.gitignore` and
  the `.gitleaks.toml` scan-skip `paths` are deliberately *different*
  sets, not two copies to keep in sync — they overlap on the
  build/venv dirs (`.venv/`, `dist/`, `build/`), but each holds paths
  the other doesn't: `.gitignore` adds `*.db` (a vault is
  path-ignored but still *scanned* by gitleaks, on purpose), while
  `.gitleaks.toml` skips `.git/` (which git never ignores via a
  rule). Don't try to unify them.

## Deliverables

- `.gitignore` — extended (not rewritten) with three grouped
  sections, each with a one-line comment naming *why*:
  - **Financial data / local vaults:** `*.db`, `*.sqlite`,
    `*.sqlite3`, plus the sidecar globs `*.db-*`, `*.sqlite-*`,
    `*.sqlite3-*` (SQLite writes `-wal` / `-shm` / `-journal`
    companion files next to a live vault; they carry the same data).
  - **Tooling caches:** `.pytest_cache/`, `.ruff_cache/`,
    `.mypy_cache/`.
  - **Packaging / build output:** `build/`, `dist/`,
    `*.egg-info/`, `*.dmg`, `*.AppImage`, `*.flatpak`,
    `.flatpak-builder/`.
- `tests/features/gitignore/` — a `spec.md` (one-paragraph test
  contract) + a test that asserts INV-1..INV-3 via `git check-ignore
  --no-index` on representative paths (see Test mechanics for why
  `--no-index` and inside-directory paths are required). INV-4 is
  already covered by the FIBR-0001 gate running `gitleaks`.

## Out of scope

- **The bespoke vault file extension.** If a later phase (the
  crypto/vault work, P03+) chooses a project-specific extension
  beyond `*.db` (e.g. a branded `.fbk`), that phase extends this
  ignore set. This item covers the generic database extensions the
  vault will use by default.
- **Compiled translation catalogs (`*.qm`).** Whether `.qm` files
  are generated-at-build (ignored) or checked in is a FIBR-0017
  decision (it owns the `lupdate` → `.ts` → `lrelease` → `.qm`
  pipeline); no translations exist yet, so nothing to ignore here.
- **Pre-commit hooks.** As in FIBR-0001, the controls are
  `.gitignore` + the `gitleaks` gate stage; a `pre-commit` hook is
  optional and not required.
- **Global (`~/.config/git/ignore`) ignores.** This item ships the
  in-repo `.gitignore` only.

## Exit criteria (Definition of Done)

1. INV-1..INV-3 pass: the `git check-ignore --no-index` test asserts
   vaults (and their sidecars), build/packaging, and tooling output
   are ignored and real source is not.
2. INV-4: `scripts/ci-local.sh` still exits 0 (the `gitleaks` stage
   green).
3. `/audit` and `/indie-review` return zero **actionable** findings
   (non-allowlisted, severity ≥ LOW per `docs/audit-allowlist.md`)
   on the same closing pass. Audit scope: `.gitignore`,
   `tests/features/gitignore/`, and any `.gitleaks.toml` change.
4. User approves closing this item.
