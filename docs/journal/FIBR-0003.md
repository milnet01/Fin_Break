## What was learned

- **The build host's glibc — not the wheels' manylinux tags — sets the
  floor.** A native freeze on this host (glibc 2.43) would not run on
  `debian:13-slim` (2.41), because the bundled `libpython` links the host
  glibc. Fix: freeze inside a controlled older-glibc container
  (`python:3.12-slim-bookworm`, ~2.36). This is the exact portability trap
  the de-risk exists to catch, found at P01 instead of P13.
- **manylinux images ship a *static* Python** with no shared `libpython`,
  which PyInstaller cannot freeze — so manylinux is the wheel-tag/floor
  concept, never the build image. `python:*-slim` provides the shared
  `libpython`.
- **`libGL`/`libEGL` are the one thing every desktop provides and no bundle
  ships** (driver-tied). Force-bundling them risks a cascade; the honest
  disposition is to bundle the *app-specific* Qt libs (glib, freetype,
  fontconfig, harfbuzz) and document `libGL` as the single runtime
  requirement — represented in the clean-room by a derived
  `debian:13-slim + libgl1 libegl1` image.
- **podman needs `--security-opt label=disable`** for bind-mount reads on
  this host, and there is **no `--env-clear` flag** (containers start with a
  clean env by default) — both surfaced as real build-run failures and were
  reconciled honestly into the spec rather than papered over.
